# ActiChat v1.4.0

## Highlights

- Rebranded the app to `ActiChat` in English-facing project/config surfaces and `动话` in Chinese user-facing surfaces, including the web title/metadata and Android app label resources.
- Replaced the old `runtime-shell` built-in flow with direct native `run` execution plus session-aware run tracking in the host/runtime pipeline.
- Upgraded `union-search` with an explicit `visit_url` webpage-reading workflow, richer direct HTML extraction, desktop-Chromium-style request handling, and improved blocked-page fallbacks.
- Added a browser-backed page-extraction path for native app builds so `visit_url` / `fetch_url` can extract webpage content through a hidden WebView, including the bundled `browser-page-extractor.js` asset.

## Packaging Notes

- The repository still treats `public/runtime-packages/python-termux-aarch64-scientific.zip` as a generated release artifact rather than a regular Git-tracked blob.
- The Android package identifier remains `com.dandwan.chatroomai` for install/update continuity even though the product branding now shows `ActiChat` / `动话`.
