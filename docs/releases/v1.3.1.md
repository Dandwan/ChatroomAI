# ChatroomAI v1.3.1

## Highlights

- Added a bundled ARM64 Python runtime to runtime management, shipped inside the APK and auto-installed on first launch without Termux, root, adb-side package installs, or any device-specific setup.
- Bundled the default scientific Python stack for built-in skill execution, including `numpy`, `pandas`, `matplotlib`, `pip`, and `lxml`, plus the supporting dependencies needed for common data and plotting tasks.
- Switched runtime process working directories to each conversation workspace so relative-path reads and writes now land in the correct per-conversation workspace for Node, Python, and future installed runtimes.
- Deleting a conversation now also deletes its corresponding workspace directory, preventing orphaned workspace data from accumulating on-device.
- Added queued skill-turn appends while a skill request is already running, plus smoother long-conversation scrolling controls in the composer area.

## Packaging Notes

- The repository tracks the Python runtime packager and metadata, but treats the generated ARM64 Python runtime ZIP as a build/release artifact instead of a regular Git blob.
- Generate the bundled Python runtime asset before Android packaging with:
  - `npm run runtime:package:python -- --output-dir public/runtime-packages`
- Android sync now fails fast if any required bundled runtime asset is missing.
