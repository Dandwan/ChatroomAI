# ActiChat v1.5.0

## Highlights

- Added a first-class `<edit>` host action for line-based text-file editing, with atomic `insert` / `delete` / `replace`, optional `expectedText`, and grouped preview snippets in host results.
- Standardized path-oriented host actions on the protocol field `location`, with `root` as the external system-path location, while keeping legacy `root` / `absolute` input compatibility for older conversations and prompts.
- Expanded `<read>` beyond `skill` and `workspace` so native builds can now inspect `home` and system absolute-path locations before editing them.
- Split the previous combined run/edit tag prompt into separate `<run>` and `<edit>` prompt settings, including dedicated global and provider-level override panels in the app settings UI.

## Packaging Notes

- System absolute-path read/edit support is implemented through the native `SkillRuntime` plugin on Android builds, not through Capacitor `Filesystem`.
- `public/runtime-packages/python-termux-aarch64-scientific.zip` continues to be treated as a generated release artifact rather than a regular Git-tracked source blob.
- The Android package identifier remains `com.dandwan.chatroomai` for install/update continuity even though the product branding now shows `ActiChat` / `动话`.
